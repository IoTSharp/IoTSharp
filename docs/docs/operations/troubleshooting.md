# 常见问题

## 页面无法访问

优先检查：

- 后端服务是否启动
- 前端是否正确连接 `/api`
- 反向代理是否转发了 WebSocket / API / 静态文件

## 首次访问总是跳到 Installer

通常是初始化没有成功落库，重点检查：

- 数据库连接
- 当前环境配置文件
- 安装提交接口返回值

## 设备在线但没有遥测

优先排查：

- 协议接入配置
- 设备或网关映射
- 规则链是否丢弃或转换了数据
- 遥测存储配置是否正确

## 告警异常增多

建议联动检查：

- 设备最新遥测
- 规则链设计
- 规则链审计

## Windows 服务启动失败

重点查看：

- 服务日志
- 安装目录权限
- 当前工作目录下的 Sqlite / 证书文件
- `ASPNETCORE_ENVIRONMENT` 是否符合预期

## 文档站点构建失败

文档目录下可先执行：

```bash
npm install
npm run build
```

再检查：

- Docusaurus 配置
- Markdown 链接
- Pages 部署环境变量
