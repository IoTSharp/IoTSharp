# Web API

IoTSharp 服务端提供了完整的 Web API，可用于接入自定义前端、脚本或第三方系统。

## 认证方式

正常调用流程通常是：

1. 登录接口获取 JWT。
2. 在后续请求中携带 `Authorization: Bearer <token>`。

## 常见接口范围

从当前代码结构看，常用接口包括：

- 账号与登录
- 设备
- 告警
- 规则链
- 租户
- 产品
- 资产
- 菜单

## 建议的调试顺序

1. 先调用登录接口。
2. 调用菜单接口确认角色和权限。
3. 再进入设备、规则链、告警等业务接口。

## 接口文档

服务端启用了 OpenAPI / Swagger UI，部署后可直接在平台内查看接口说明。
