# 协议接入

IoTSharp 面向设备和网关提供多种接入方式。

## 支持方向

- HTTP
- MQTT
- CoAP
- 网关协议扩展

## MQTT

适合：

- 持续在线设备
- 低延迟遥测上报
- 订阅式控制

平台中常见配套能力：

- 遥测上报
- 属性同步
- 证书下载
- TLS / X509

## HTTP

适合：

- 简单设备
- 周期性上报
- 与第三方系统对接

## 网关协议

当前代码中已经包含 Modbus、OPC UA 等网关设计面板，这说明 IoTSharp 支持通过网关把非原生协议设备统一映射为平台对象。

## 推荐实践

- 简单场景优先 HTTP。
- 长连接设备优先 MQTT。
- 工业现场优先通过网关归一化接入。
